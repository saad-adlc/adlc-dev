# WS2/WS3/WS4 — gh-aw Agentic Spine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development or executing-plans. Steps use checkbox (`- [ ]`) syntax.
> **Supersedes** design-doc G6 per verified gh-aw v0.79.8 mechanics (see the design doc's 2026-06-19 addendum).

**Goal:** Port `adlc-generate` and `adlc-iterate` to gh-aw as the primary engine — Spec-Kit-structured generation (spec/plan/tasks + TDD) and self-healing iteration, both Foundry-billed via `engine: claude`, with the WS0 deny hook mounted and gh-aw-native enforcement as the deterministic guardrail — and park the hand-rolled workflows as fallback.

**Architecture:** A deterministic pre-agent script (`adlc-prep.sh`) does the byte-identical slug, branch checkout, workspace scaffold, npm install, status.json, and hook/standards mounting. The gh-aw agent then writes spec/plan/tasks + TDD code inside `workspaces/<slug>/` and commits. `safe-outputs.create-pull-request` opens the PR (never auto-merged). Iterate mirrors this with `push-to-pull-request-branch` and a deterministic 3-iteration cap.

**Tech Stack:** gh-aw v0.79.8 (pinned), `engine: claude` (`claude` CLI, Foundry base-URL), bash + jq, Node 20 (CI only).

## Global Constraints

- **Repo/branch:** all paths in `adlc-dev/`, committed on `ws-doc9-impl`. WS0 (`adlc-standards`) must be merged to `adlc-standards` `main` before the live smoke test (CI clones it).
- **Verified gh-aw mechanics (v0.79.8 source):**
  - Custom pre-agent `steps:` are supported and run outside the sandbox.
  - `create-pull-request` is **commit-based** (bundles the agent's commits); the agent **must commit**. Branch = the checked-out branch; `preserve-branch-name: true` keeps it exact; `allowed-branches`, `title-prefix`, `labels`, `draft` are policy.
  - **Protected Files** are **basename-matched** and (Claude engine) include `.github/`, `.claude/`, `CLAUDE.md`, `AGENTS.md`, `CODEOWNERS`, `README.md`, `package.json`, `package-lock.json`. The scaffold's `package.json`/`package-lock.json` MUST be `exclude`d; cloned standards + mounted `.claude/` MUST be kept out of the agent's commit (`.git/info/exclude`).
  - The Claude engine runs under `--permission-mode acceptEdits` (NOT `bypassPermissions`); `--allowed-tools` is enforced; PreToolUse hooks **do** execute (verify-loads required — settings-file resolution path is empirical).
  - `push-to-pull-request-branch` pushes the agent's commits to the triggering PR's head branch; requires a PR-context event; fork PRs unsupported.
  - Foundry: `engine.env.ANTHROPIC_BASE_URL` + `ANTHROPIC_API_KEY` GitHub secret (auto-wired, NOT in `engine.env`) + host in `network.allowed`. `model: claude-sonnet-4-6`.
- **Slug invariant (HANDOFF §3/§12.1):** `adlc-prep.sh`'s slug must be **byte-identical** to the hand-rolled `adlc-generate.yml` algorithm. Verified against real pipeline branch names.
- **Deterministic cap (design G8):** iterate enforces ≤3 in a pre-agent step, never the prompt.
- **Local-vs-CI verification:** `gh aw compile` and `npm` are NOT available locally. Tasks mark each check as **[local]** (runnable now in bash) or **[CI/you]** (run by you / in Actions). The slug logic is `[local]` TDD; the `.md` workflows are authored here and `[CI/you]`-compiled.

- [ ] **Step 3: Write `adlc-prep.sh` (GREEN)** — create `.github/scripts/adlc-prep.sh`:

```bash
#!/usr/bin/env bash
# ADLC deterministic pre-agent prep for the gh-aw generate workflow.
# Slug is BYTE-IDENTICAL to the hand-rolled adlc-generate.yml (HANDOFF §3/§12.1).
set -uo pipefail

adlc_slug() {  # <issue> <title> -> "issue-<N>-<kebab>"
  local issue="$1" title="$2" kebab
  kebab=$(printf '%s' "$title" | LC_ALL=C tr '[:upper:]' '[:lower:]' \
    | LC_ALL=C sed -E 's/[^a-z0-9]+/-/g; s/^-+//; s/-+$//')
  if [ "${#kebab}" -gt 40 ]; then
    kebab="${kebab:0:40}"
    kebab=$(printf '%s' "$kebab" | sed -E 's/-[^-]*$//; s/-+$//')
  fi
  [ -z "$kebab" ] && kebab="feature"
  printf 'issue-%s-%s' "$issue" "$kebab"
}

# Allow tests to source functions without executing main.
case "${1:-}" in --source-only) return 0 2>/dev/null || exit 0 ;; esac

main() {
  : "${ISSUE_NUMBER:?}"; : "${ISSUE_TITLE:?}"
  local slug branch ws
  slug="$(adlc_slug "$ISSUE_NUMBER" "$ISSUE_TITLE")"
  branch="feature/${slug}"
  ws="workspaces/${slug}"

  git config user.name "adlc-agent"
  git config user.email "adlc-agent@saad-adlc.com"
  git checkout -b "$branch"

  # --- scaffold workspaces/<slug>/ (identical to hand-rolled) ---
  mkdir -p "${ws}/src"
  cat > "${ws}/package.json" <<EOF
{
  "name": "${slug}",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "typecheck": "tsc --noEmit",
    "lint": "eslint . --max-warnings 0",
    "test": "vitest run",
    "test:coverage": "vitest run --coverage",
    "ci": "npm run typecheck && npm run lint && npm run test:coverage"
  },
  "dependencies": { "react": "^18.3.1", "react-dom": "^18.3.1" },
  "devDependencies": {
    "@eslint/js": "^9.15.0",
    "@testing-library/jest-dom": "^6.4.0",
    "@testing-library/react": "^16.0.1",
    "@types/react": "^18.3.12",
    "@types/react-dom": "^18.3.1",
    "@vitejs/plugin-react": "^4.3.3",
    "@vitest/coverage-v8": "^2.1.5",
    "eslint": "^9.15.0",
    "globals": "^15.12.0",
    "jsdom": "^25.0.1",
    "typescript": "^5.6.3",
    "typescript-eslint": "^8.15.0",
    "vite": "^5.4.11",
    "vitest": "^2.1.5"
  }
}
EOF
  cat > "${ws}/tsconfig.json" <<'EOF'
{
  "compilerOptions": {
    "target": "ES2020", "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"], "module": "ESNext",
    "skipLibCheck": true, "moduleResolution": "bundler",
    "resolveJsonModule": true, "isolatedModules": true,
    "moduleDetection": "force", "noEmit": true, "jsx": "react-jsx",
    "strict": true, "noUnusedLocals": true, "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true, "types": ["vitest/globals"]
  },
  "include": ["src"]
}
EOF
  cat > "${ws}/vite.config.ts" <<'EOF'
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';
export default defineConfig({
  plugins: [react()],
  test: {
    globals: true, environment: 'jsdom', setupFiles: ['./src/test-setup.ts'],
    coverage: {
      provider: 'v8', reporter: ['text', 'lcov', 'json-summary'],
      thresholds: { lines: 80, functions: 80, branches: 80, statements: 80 },
      exclude: ['**/main.tsx', '**/*.config.*', '**/dist/**', '**/coverage/**'],
    },
  },
});
EOF
  cat > "${ws}/eslint.config.mjs" <<'EOF'
import js from '@eslint/js';
import tseslint from 'typescript-eslint';
import globals from 'globals';
export default tseslint.config(
  { ignores: ['dist', 'coverage', '*.config.ts', 'eslint.config.mjs'] },
  js.configs.recommended,
  ...tseslint.configs.recommended,
  { files: ['**/*.{ts,tsx}'], languageOptions: { globals: { ...globals.browser } } },
);
EOF
  cat > "${ws}/index.html" <<EOF
<!doctype html>
<html lang="en"><head><meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>${slug}</title></head>
<body><div id="root"></div><script type="module" src="/src/main.tsx"></script></body></html>
EOF
  printf "import '@testing-library/jest-dom';\n" > "${ws}/src/test-setup.ts"
  cat > "${ws}/src/main.tsx" <<'EOF'
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';
createRoot(document.getElementById('root')!).render(<StrictMode><App /></StrictMode>);
EOF
  printf '/** App — root component. */\nexport default function App() {\n  return <div id="app" />;\n}\n' > "${ws}/src/App.tsx"

  ( cd "$ws" && npm install )   # commits lockfile via the agent later

  # --- mount standards + deny hook, kept OUT of the agent's commit ---
  git clone --depth 1 "https://x-access-token:${STANDARDS_TOKEN}@github.com/${STANDARDS_REPO}.git" .adlc-standards
  mkdir -p .claude/hooks
  cp .adlc-standards/hooks/pretooluse-deny.sh .claude/hooks/pretooluse-deny.sh
  chmod +x .claude/hooks/pretooluse-deny.sh
  cp .adlc-standards/hooks/settings.template.json .claude/settings.json
  cp .adlc-standards/constitution.md ./constitution.md
  cp -r .adlc-standards/steering ./steering
  # never commit the mounted standards / hook config (Protected Files + cleanliness)
  printf '%s\n' '.adlc-standards/' '.claude/' 'constitution.md' 'steering/' >> .git/info/exclude

  # --- status.json: scaffolding ---
  mkdir -p "${ws}/.adlc"
  cat > "${ws}/.adlc/status.json" <<JSON
{ "stage": "scaffolding", "iteration": 0, "pr": null, "preview_url": null,
  "detail": "Workspace created, dependencies installed",
  "updated": "$(date -u +%Y-%m-%dT%H:%M:%SZ)" }
JSON

  # --- export for the agent + safe-output ---
  {
    echo "ADLC_BRANCH=${branch}"
    echo "ADLC_WORKSPACE=${ws}"
    echo "ADLC_SLUG=${slug}"
  } >> "$GITHUB_ENV"
  echo "Prepared ${branch} / ${ws}"
}

main "$@"
```

# ADLC Generate (Spec-Kit structured, TDD)

You implement a GitHub issue as a React+TypeScript+Vite+Vitest app. The workspace is
already scaffolded and dependencies installed by the prep step. Your branch is
already checked out (`$ADLC_BRANCH`); your workspace is `$ADLC_WORKSPACE`.

## Hard rules (enforced by `.claude/hooks/pretooluse-deny.sh` + gh-aw)
- Work ONLY inside `$ADLC_WORKSPACE`. Never touch `.github/`, root config, other
  workspaces, `.claude/`, `constitution.md`, or `steering/`.
- Do NOT run `npm install` (already done). Do NOT push or merge.

## Steps
1. Read `constitution.md`, `steering/approved-stack.md`, `steering/compliance-rules.md`.
2. Read the issue (title + body below) — it is the intent.
3. Write the audit-trail artifacts in `$ADLC_WORKSPACE/`:
   - `spec.md` — what we're building, acceptance criteria (checklist), inputs/outputs, test requirements (≥80% lines), assumptions, out-of-scope.
   - `plan.md` — file map + the interface each file exposes.
   - `tasks.md` — TDD task list, each mapped to an acceptance criterion.
4. Implement TDD inside `$ADLC_WORKSPACE/src/` — tests first (`*.test.tsx`,
   `toBeInTheDocument()` not `toBeDefined()`), then the minimal component; update `src/App.tsx` to mount it.
5. Analyze-style self-check: every acceptance criterion maps to a task and a test;
   no placeholders; only the approved stack; no banned packages.
6. Run `cd $ADLC_WORKSPACE && npm run ci` and fix until green.
7. Commit ONLY your workspace: `git add "$ADLC_WORKSPACE" && git commit -m "feat($ADLC_SLUG): implement issue #${{ github.event.issue.number }}"`.

## Output
- Emit a **create-pull-request** safe output: title = the feature in plain words
  (the `feat: ` prefix is added), body = summary + `Closes #${{ github.event.issue.number }}` + a changes/tests table + a link to `spec.md`.
- Emit one **add-comment** on the issue: a plain-language "what was built" summary (no file paths/error codes).

on:
  pull_request:
    types: [labeled]
    names: [adlc-iterate]
  pull_request_review:
    types: [submitted]
  issue_comment:
    types: [created]
  workflow_run:
    workflows: ["ADLC — CI"]
    types: [completed]
    branches: ["feature/issue-*"]

permissions:
  contents: read
  pull-requests: read
  issues: read

engine:
  id: claude
  model: claude-sonnet-4-6
  env:
    ANTHROPIC_BASE_URL: https://orix-adastra-adlc.services.ai.azure.com/anthropic

network:
  allowed:
    - defaults
    - orix-adastra-adlc.services.ai.azure.com

tools:
  edit:
  bash: [":*"]

steps:
  - name: Checkout PR head
    uses: actions/checkout@v6
    with:
      token: ${{ secrets.ADLC_AGENT_TOKEN }}
      fetch-depth: 0
  - name: Setup Node 20
    uses: actions/setup-node@v4
    with:
      node-version: '20'

    env:
      STANDARDS_REPO: saad-adlc/adlc-standards

      set -uo pipefail
      git fetch origin main

      if [ "$N" -ge 3 ]; then
        echo "ADLC_CAPPED=true" >> "$GITHUB_ENV"
      else
        echo "ADLC_CAPPED=false" >> "$GITHUB_ENV"
        echo "ADLC_NEXT=$((N + 1))" >> "$GITHUB_ENV"
        git clone --depth 1 "https://x-access-token:${STANDARDS_TOKEN}@github.com/${STANDARDS_REPO}.git" .adlc-standards
        mkdir -p .claude/hooks
        cp .adlc-standards/hooks/pretooluse-deny.sh .claude/hooks/pretooluse-deny.sh
        chmod +x .claude/hooks/pretooluse-deny.sh

        cp .adlc-standards/constitution.md ./constitution.md
        printf '%s\n' '.adlc-standards/' '.claude/' 'constitution.md' >> .git/info/exclude
        # ADLC_WORKSPACE = the single workspaces/<slug>/ dir changed on this branch
        WS=$(git diff --name-only origin/main..HEAD | grep -oE '^workspaces/[^/]+' | sort -u | head -1)
        echo "ADLC_WORKSPACE=${WS}" >> "$GITHUB_ENV"

safe-outputs:
  push-to-pull-request-branch:
    target: triggering
    required-labels: [adlc-generated]
    if-no-changes: warn
    max: 1
    protected-files:
      policy: fallback-to-issue
      exclude: ["package.json", "package-lock.json", "README.md"]
  add-comment:
    max: 1

# ADLC Iterate (self-healing, max 3)

You fix an existing ADLC PR. If `$ADLC_CAPPED` is `true`, STOP: emit a single
add-comment saying the 3-iteration cap is reached and a human must intervene, and
make NO code changes.

Otherwise (`$ADLC_CAPPED` is `false`, attempt `$ADLC_NEXT` of 3):
- Work ONLY inside `$ADLC_WORKSPACE`. The deny hook + gh-aw enforce this. Do NOT
  run `npm install`, push, or merge.
- Read `constitution.md` first. Determine the failure from the triggering event:
  a failing **ADLC — CI** run, a changes-requested review, or a `/adlc-iterate:`
  issue comment (apply the requested change exactly and minimally).
- Make the **minimal** fix — no refactors, no new dependencies, no deleted/weakened tests.
- Run `cd $ADLC_WORKSPACE && npm run ci` until green.
- Append a dated entry to `$ADLC_WORKSPACE/spec.md` `## Change log` in plain language.
- Commit ONLY your workspace: `git add "$ADLC_WORKSPACE" && git commit -m "fix: iteration-$ADLC_NEXT"`.

## Output
- Emit a **push-to-pull-request-branch** safe output (pushes your fix commit to the PR head; CI re-runs).
- Emit one **add-comment**: "attempt N/3 pushed" in plain language.
