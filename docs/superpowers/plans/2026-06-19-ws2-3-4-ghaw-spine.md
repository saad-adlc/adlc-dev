# WS2/WS3/WS4 — gh-aw Agentic Spine Implementation Plan (rev. 4)

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development or executing-plans. Steps use checkbox (`- [ ]`) syntax.
> **Supersedes** design-doc G6 per verified gh-aw v0.79.8 mechanics (design-doc addendum). **Rev. 2** baked in grill decisions Q1–Q3; **rev. 3** applied the Opus review (Fallback B default, `blocked` policy); **rev. 4** replaces parking with an **engine-coexistence model** — both engines always wired, `ADLC_ENGINE` override + **2-strike auto-fallback** to hand-rolled.

**Goal:** Port `adlc-generate` and `adlc-iterate` to gh-aw as the primary engine — Spec-Kit-templated generation (spec/plan/tasks + TDD) and self-healing iteration, Foundry-billed via `engine: claude`, WS0 deny hook mounted, gh-aw-native enforcement deterministic — while the **hand-rolled workflows stay fully wired as an always-ready fallback** (never parked): an `ADLC_ENGINE` repo-variable override plus a plain-Actions controller that auto-falls-back after 2 gh-aw strikes.

**Architecture:** A deterministic pre-agent script (`adlc-prep.sh`) does the byte-identical slug, local branch checkout, scaffold (Fallback B: **uncommitted** — the agent commits it), npm install, and mounts the deny hook + Spec-Kit templates. The gh-aw agent fills the mounted templates and writes TDD code inside `workspaces/<slug>/`, then commits; `create-pull-request` opens the PR (never auto-merged); `adlc-status-pr.yml` writes status `pr-open`. **Iterate stays the proven hand-rolled workflow** — gh-aw strict mode bans the `contents: write` its deterministic cap-3 + status pushes need, so it is NOT ported; it remains always-active (live status via its plain-workflow `contents: write`), serving PRs from either generator. **Engine coexistence (generate only):** the gh-aw generate and the hand-rolled generate both stay wired to the real triggers, made mutually exclusive by an `ADLC_ENGINE` `if:`-guard; a plain-Actions **fail-over controller** retries gh-aw once and, on the **2nd strike** (or agent-BLOCKED), routes the issue to the hand-rolled equivalent by adding an **`adlc-fallback` label** (a real `issues: labeled` event, so the proven workflow's internals are untouched) — propagating the error via the issue-comment thread + a run-context artifact.

**Tech Stack:** gh-aw v0.79.8 (pinned), `engine: claude` (Foundry base-URL), bash + jq, Node 20 (CI only), vendored Spec-Kit v0.10.3 templates.

## Global Constraints

- **Repo/branch:** all paths in `adlc-dev/`, committed on `ws-doc9-impl`. WS0 (`adlc-standards`) must be merged to `adlc-standards` `main` before the live smoke test (CI clones it).
- **Verified gh-aw mechanics (v0.79.8 source):**
  - Custom pre-agent `steps:` + `pre-agent-steps:` + `post-steps:` are supported (run outside the sandbox, with the job's token/permissions).
  - `create-pull-request` is **commit-based**: it bundles the agent's commits (`origin/<branch>..<branch>`) and a separate `safe_outputs` job applies the bundle to `refs/heads/<branch>` and opens the PR. The agent **must commit**, and the prep step must **not** push the branch early (early commits would be excluded from the bundle → scaffold-less PR; this is why generate uses Fallback B). Branch = the checked-out branch; `preserve-branch-name: true` keeps it exact; `allowed-branches`, `title-prefix`, `labels`, `draft` are policy.
  - **Protected Files** are **basename-matched** (Claude engine set includes `.github/`, `.claude/`, `CLAUDE.md`, `AGENTS.md`, `CODEOWNERS`, `README.md`, `package.json`, `package-lock.json`, …). The scaffold's `package.json`/`package-lock.json` MUST be `exclude`d; cloned standards + mounted `.claude/` + Spec-Kit templates MUST be kept out of the agent's commit (`.git/info/exclude` + scoped `git add`).
  - Claude engine runs under `--permission-mode acceptEdits` (NOT bypassPermissions); `--allowed-tools` enforced; PreToolUse hooks **do** execute (verify-loads in Task 7).
  - `push-to-pull-request-branch` pushes the agent's commits to the triggering PR's head; requires a PR-context event; fork PRs unsupported.
  - Foundry: `engine.env.ANTHROPIC_BASE_URL` + `ANTHROPIC_API_KEY` GitHub secret (auto-wired, NOT in `engine.env`) + host in `network.allowed`. `model: claude-sonnet-4-6`.
  - **Strict mode:** every `actions/checkout` in the agent job MUST set `persist-credentials: false` — otherwise the token is written to `.git/config` and leaks into the sandboxed agent. `gh aw compile` hard-errors without it. (Safe here: generate never pushes; iterate pushes via an explicit token URL; `git fetch origin` works anonymously on the public repo.)
- **Live status.json (grill Q1 → revised after source review):** the early-push-to-feature-branch mechanism is **source-proven incompatible** with `create-pull-request` (gh-aw bundles `origin/<branch>..<branch>`, so early-pushed commits are excluded → a scaffold-less PR). So **generate uses Fallback B**: the prep step writes status `scaffolding` **uncommitted**, and the agent commits it together with its work; `adlc-status-pr.yml` sets `pr-open` (carries PR#); terminal stages come from `adlc-preview`/`adlc-signals`. The generate **agent job is `contents: read`** (no early push). **Iterate keeps live status** (`iterating`/`escalated` pushed by its steps) because `push-to-pull-request-branch` *appends* to the existing PR branch instead of rebuilding it. Live *pre-PR* narration (first ~15 min) is deferred to the **WS7 decoupled issue-comment channel**; native GitHub events (PR/checks/Pages) carry PR→preview liveness regardless. The HANDOFF §5/§12 channel is preserved in shape, fed differently per phase.
- **Spec-Kit fidelity (grill Q2):** the prep step mounts `vendor/spec-kit/templates/{spec,plan,tasks}-template.md`; the agent fills them and may adapt/omit fields for small MVP features.
- **Bash scope (grill Q3):** moderate `--allowed-tools` allowlist (`npm/npx/node`, `git:*`, `ls/cat/mkdir/mv/cp/sed/grep`); `curl`/`wget` excluded (firewall + deny hook cover the rest).
- **Slug invariant (HANDOFF §3/§12.1):** `adlc_slug` byte-identical to the hand-rolled algorithm (locally tested).
- **Deterministic cap (design G8):** iterate enforces ≤3 in a pre-agent step, never the prompt.
- **Iterate is hand-rolled-only (not ported):** gh-aw strict mode forbids the `contents: write` that iterate's deterministic cap-3 + `iterating`/`escalated` status pushes require (writes must go via safe-outputs, which can't enforce a cap deterministically — would violate design G8). So `adlc-iterate.yml` stays the proven engine, **always active** (no `ADLC_ENGINE` guard), handling PRs from either generator. The `ADLC_ENGINE` coexistence + 2-strike fail-over below apply to **generate only**.
- **Engine coexistence (generate) — `ADLC_ENGINE` override + 2-strike auto-fallback (NO parking):** the hand-rolled `adlc-generate.yml` and the gh-aw `adlc-generate.lock.yml` are **both always wired** to the real triggers; an `ADLC_ENGINE` repo-variable `if:`-guard makes them mutually exclusive (no double-run). Default `gh-aw`: gh-aw runs, hand-rolled skip (job `if:` false). `ADLC_ENGINE=legacy`: gh-aw skips, hand-rolled run directly on the label — the **proven path, one variable flip, instant** (no workflow edits). A plain-Actions **fail-over controller** (`adlc-failover.yml`, engine-agnostic so it never depends on gh-aw) watches gh-aw generate `workflow_run` failures: a **transient** failure retries gh-aw once (re-applies the `adlc-generate` label); the **2nd strike OR agent-BLOCKED/clarification** adds an **`adlc-fallback` label** that routes the issue to the hand-rolled generator. The hand-rolled never starts mid-pipeline (the controller fires only after a run *concludes*) and never double-runs (mutually-exclusive `ADLC_ENGINE` guards + gh-aw's `names: [adlc-generate]` filter ignores `adlc-fallback`). **Error propagation:** the controller reads the failed run's **agent-log artifact** (error summary) + a **run-context artifact** (`adlc-context.json {issue}`, for run→issue mapping), records each strike as a **structured issue comment** (durable audit + human-visible + the next agent re-reads it), and counts strikes from those comments.
- **Local-vs-CI:** `gh aw compile` and `npm` are NOT available locally. Steps are tagged **[local]** (runnable now) or **[CI/you]** (you run / Actions runs). Only `adlc_slug` is unit-tested locally; `.md` workflows are authored here, compiled by you.

```bash
#!/usr/bin/env bash
# ADLC deterministic pre-agent prep for the gh-aw generate workflow.
# Slug is BYTE-IDENTICAL to the hand-rolled adlc-generate.yml (HANDOFF §3/§12.1).
set -uo pipefail

adlc_slug() {  # <issue> <title> -> "issue-<N>-<kebab>"
  local issue="$1" title="$2" kebab
  kebab=$(printf '%s' "$title" | LC_ALL=C tr '[:upper:]' '[:lower:]' \
    | LC_ALL=C sed -E 's/[^a-z0-9]+/-/g; s/^-+//; s/-+$//')
  if [ "${#kebab}" -gt 40 ]; then
    kebab="${kebab:0:40}"; kebab=$(printf '%s' "$kebab" | sed -E 's/-[^-]*$//; s/-+$//')
  fi
  [ -z "$kebab" ] && kebab="feature"
  printf 'issue-%s-%s' "$issue" "$kebab"
}

write_status() {  # <ws> <stage> <detail> [pr]
  local ws="$1" stage="$2" detail="$3" pr="${4:-null}"
  mkdir -p "${ws}/.adlc"
  cat > "${ws}/.adlc/status.json" <<JSON
{ "stage": "${stage}", "iteration": 0, "pr": ${pr}, "preview_url": null,
  "detail": "${detail}", "updated": "$(date -u +%Y-%m-%dT%H:%M:%SZ)" }
JSON
}

# Allow tests to source functions without running main.
case "${1:-}" in --source-only) return 0 2>/dev/null || exit 0 ;; esac

main() {
  : "${ISSUE_NUMBER:?}"; : "${ISSUE_TITLE:?}"; : "${STANDARDS_REPO:?}"; : "${STANDARDS_TOKEN:?}"
  local slug branch ws
  slug="$(adlc_slug "$ISSUE_NUMBER" "$ISSUE_TITLE")"
  branch="feature/${slug}"; ws="workspaces/${slug}"

  git config user.name "adlc-agent"
  git config user.email "adlc-agent@saad-adlc.com"
  git checkout -b "$branch"

  # --- scaffold workspaces/<slug>/ (identical to hand-rolled) ---
  mkdir -p "${ws}/src"
  cat > "${ws}/package.json" <<EOF
{
  "name": "${slug}", "private": true, "version": "0.0.0", "type": "module",
  "scripts": {
    "dev": "vite", "build": "vite build", "typecheck": "tsc --noEmit",
    "lint": "eslint . --max-warnings 0", "test": "vitest run",
    "test:coverage": "vitest run --coverage",
    "ci": "npm run typecheck && npm run lint && npm run test:coverage"
  },
  "dependencies": { "react": "^18.3.1", "react-dom": "^18.3.1" },
  "devDependencies": {
    "@eslint/js": "^9.15.0", "@testing-library/jest-dom": "^6.4.0",
    "@testing-library/react": "^16.0.1", "@types/react": "^18.3.12",
    "@types/react-dom": "^18.3.1", "@vitejs/plugin-react": "^4.3.3",
    "@vitest/coverage-v8": "^2.1.5", "eslint": "^9.15.0", "globals": "^15.12.0",
    "jsdom": "^25.0.1", "typescript": "^5.6.3", "typescript-eslint": "^8.15.0",
    "vite": "^5.4.11", "vitest": "^2.1.5"
  }
}
EOF
  cat > "${ws}/tsconfig.json" <<'EOF'
{
  "compilerOptions": {
    "target": "ES2020", "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"], "module": "ESNext",
    "skipLibCheck": true, "moduleResolution": "bundler", "resolveJsonModule": true,
    "isolatedModules": true, "moduleDetection": "force", "noEmit": true,
    "jsx": "react-jsx", "strict": true, "noUnusedLocals": true,
    "noUnusedParameters": true, "noFallthroughCasesInSwitch": true,
    "types": ["vitest/globals"]
  },
  "include": ["src"]
}
EOF
  cat > "${ws}/vite.config.ts" <<'EOF'
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';
export default defineConfig({
  plugins: [react()],
  test: {
    globals: true, environment: 'jsdom', setupFiles: ['./src/test-setup.ts'],
    coverage: { provider: 'v8', reporter: ['text', 'lcov', 'json-summary'],
      thresholds: { lines: 80, functions: 80, branches: 80, statements: 80 },
      exclude: ['**/main.tsx', '**/*.config.*', '**/dist/**', '**/coverage/**'] },
  },
});
EOF
  cat > "${ws}/eslint.config.mjs" <<'EOF'
import js from '@eslint/js';
import tseslint from 'typescript-eslint';
import globals from 'globals';
export default tseslint.config(
  { ignores: ['dist', 'coverage', '*.config.ts', 'eslint.config.mjs'] },
  js.configs.recommended, ...tseslint.configs.recommended,
  { files: ['**/*.{ts,tsx}'], languageOptions: { globals: { ...globals.browser } } },
);
EOF
  cat > "${ws}/index.html" <<EOF
<!doctype html>
<html lang="en"><head><meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>${slug}</title></head>
<body><div id="root"></div><script type="module" src="/src/main.tsx"></script></body></html>

  printf "import '@testing-library/jest-dom';\n" > "${ws}/src/test-setup.ts"
  cat > "${ws}/src/main.tsx" <<'EOF'
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';
createRoot(document.getElementById('root')!).render(<StrictMode><App /></StrictMode>);
EOF
  printf '/** App — root component. */\nexport default function App() {\n  return <div id="app" />;\n}\n' > "${ws}/src/App.tsx"

  ( cd "$ws" && npm install )

  # --- mount standards + deny hook + Spec-Kit templates, kept OUT of the agent's commit ---
  git clone --depth 1 "https://x-access-token:${STANDARDS_TOKEN}@github.com/${STANDARDS_REPO}.git" .adlc-standards
  mkdir -p .claude/hooks .speckit
  cp .adlc-standards/hooks/pretooluse-deny.sh .claude/hooks/pretooluse-deny.sh
  chmod +x .claude/hooks/pretooluse-deny.sh
  cp .adlc-standards/hooks/settings.template.json .claude/settings.json
  cp .adlc-standards/constitution.md ./constitution.md
  cp -r .adlc-standards/steering ./steering
  cp .adlc-standards/vendor/spec-kit/templates/spec-template.md  .speckit/spec-template.md
  cp .adlc-standards/vendor/spec-kit/templates/plan-template.md  .speckit/plan-template.md
  cp .adlc-standards/vendor/spec-kit/templates/tasks-template.md .speckit/tasks-template.md
  printf '%s\n' '.adlc-standards/' '.claude/' 'constitution.md' 'steering/' '.speckit/' >> .git/info/exclude

  # --- status scaffolding, written but LEFT UNCOMMITTED (Fallback B = default) ---
  # The agent's single `git add "$ADLC_WORKSPACE"` will stage the scaffold + this
  # status.json + its own src into ONE agent-authored commit, which is guaranteed
  # to be in the create-pull-request bundle. We do NOT push the branch here:
  # gh-aw bundles origin/<branch>..<branch>, so any early-pushed commit would be
  # EXCLUDED from the PR (verified: generate_git_bundle.cjs). No early push, no
  # branch collision, no recreate-ref rename risk.
  write_status "$ws" "scaffolding" "Workspace created, dependencies installed"

  { echo "ADLC_BRANCH=${branch}"; echo "ADLC_WORKSPACE=${ws}"; echo "ADLC_SLUG=${slug}"; } >> "$GITHUB_ENV"
  echo "Prepared ${branch} / ${ws} (scaffold uncommitted; agent commits it)"
}

main "$@"
```

````markdown
# ADLC Generate (Spec-Kit templated, TDD)

You implement a GitHub issue as a React+TypeScript+Vite+Vitest app. The workspace
`$ADLC_WORKSPACE` is already scaffolded (deps installed) on branch `$ADLC_BRANCH`.

## Hard rules (enforced by `.claude/hooks/pretooluse-deny.sh`, Protected Files, firewall)
- Work ONLY inside `$ADLC_WORKSPACE`. Never touch `.github/`, root config, other
  workspaces, `.claude/`, `constitution.md`, `steering/`, `.speckit/`, `.adlc-standards/`.
- Do NOT run `npm install`. Do NOT push or merge. Do NOT edit `.adlc/status.json`.

## Steps
1. Read `constitution.md`, `steering/approved-stack.md`, `steering/compliance-rules.md`.
````

### Task 6: Fail-over controller (`adlc-failover.yml`)

**Files:** Create `.github/workflows/adlc-failover.yml`

**Rationale:** plain Actions (engine-agnostic — the fallback path never depends on gh-aw). On a gh-aw generate failure: retry gh-aw **once** (re-apply `adlc-generate`); on the **2nd strike or agent-BLOCKED**, add the **`adlc-fallback` label** to route to the hand-rolled generator. Strikes/errors propagate via structured issue comments + the run-context artifact.

- [ ] **Step 1: Author** — create `.github/workflows/adlc-failover.yml`:

```yaml
name: ADLC — Fail-over Controller
on:
  workflow_run:
    workflows: ["ADLC Generate"]      # MUST match the gh-aw generate workflow's compiled name:
    types: [completed]

permissions:
  actions: read       # download the failed run's artifacts/logs
  contents: read
  issues: write       # post strike comments

jobs:
  failover:
    # Only on a gh-aw generate FAILURE, and only while gh-aw is the active engine.
    if: github.event.workflow_run.conclusion == 'failure' && vars.ADLC_ENGINE != 'legacy'
    runs-on: ubuntu-latest
    steps:
      - name: Download run-context artifact (run -> issue)
        uses: actions/download-artifact@v4
        with:
          name: adlc-context
          run-id: ${{ github.event.workflow_run.id }}
          github-token: ${{ secrets.ADLC_AGENT_TOKEN }}
          path: ctx
      - name: Decide + propagate
        env:
          GH_TOKEN: ${{ secrets.ADLC_AGENT_TOKEN }}
          REPO: ${{ github.repository }}
          RUN_ID: ${{ github.event.workflow_run.id }}
        run: |
          set -uo pipefail
          ISSUE="$(jq -r '.issue' ctx/adlc-context.json)"
          [ -n "$ISSUE" ] && [ "$ISSUE" != "null" ] || { echo "no issue in context; abort"; exit 0; }

          # Error summary from the failed run's logs (best-effort; never fail the controller).
          gh run view "$RUN_ID" --repo "$REPO" --log-failed > run.log 2>/dev/null || true
          ERR="$(tail -c 1200 run.log | tr '\n\r"' '   ')"

          # Classify: agent BLOCKED / needs-clarification => do NOT retry (bad input won't improve).
          BLOCKED=false
          grep -qiE 'BLOCKED|NEEDS_CONTEXT|clarif|ambiguous' run.log && BLOCKED=true

          # Strike count = number of prior structured attempt comments on the issue.
          N="$(gh issue view "$ISSUE" --repo "$REPO" --json comments \
                 --jq '[.comments[].body | select(startswith("<!-- adlc-attempt"))] | length')"
          STRIKE=$((N + 1))

          BODY="$(printf '<!-- adlc-attempt n=%s engine=gh-aw conclusion=failure run=%s -->\n⚠️ ADLC generate (gh-aw) attempt %s failed. Summary: %s' "$STRIKE" "$RUN_ID" "$STRIKE" "${ERR:0:280}")"
          gh issue comment "$ISSUE" --repo "$REPO" --body "$BODY"

          if [ "$BLOCKED" = "true" ] || [ "$STRIKE" -ge 2 ]; then
            if [ "$BLOCKED" = "true" ]; then REASON="agent blocked / needs clarification"; else REASON="${STRIKE} strikes"; fi
            gh issue comment "$ISSUE" --repo "$REPO" --body "$(printf '↩️ Falling back to the proven hand-rolled generator (%s).' "$REASON")"
            # Ensure the routing label exists, then apply it -> fires the hand-rolled (real issue event).
            gh label create adlc-fallback --repo "$REPO" --color B60205 \
              --description "ADLC: route this issue to the hand-rolled generator" 2>/dev/null || true
            gh issue edit "$ISSUE" --repo "$REPO" --add-label adlc-fallback
          else
            # Transient: retry gh-aw by re-applying the label (re-fires its labeled trigger).
            gh issue edit "$ISSUE" --repo "$REPO" --remove-label adlc-generate || true
            sleep 3
            gh issue edit "$ISSUE" --repo "$REPO" --add-label adlc-generate
          fi
```

- [ ] **Step 2: Commit** `[local]` (plain YAML; no compile):
```bash

git commit -m "feat(ci): fail-over controller (2-strike gh-aw -> hand-rolled, engine-agnostic)"
```

> `workflows: ["ADLC Generate"]` must match the gh-aw generate `name:` (set explicitly in T3). The `adlc-fallback` label routes to the hand-rolled (T7); the controller creates the label if missing. gh-aw's `names: [adlc-generate]` filter means `adlc-fallback` never triggers gh-aw → no double-run. 2 strikes = 1 retry then fall back (each gh-aw attempt is a full Foundry run, so the cap is deliberately low). The controller is **generate-only**; iterate is hand-rolled (Task 4), so there is no gh-aw-iterate failover — the hand-rolled iterate's deterministic content cap-3 (auto-fix → human escalation) serves PRs from either generator, unchanged.
